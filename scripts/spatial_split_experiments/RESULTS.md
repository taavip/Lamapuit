# Spatial Split Experiments — Results

Generated by running `run_experiments.py` against
`output/model_search_v4/prepared/labels_curated_v4` (17 403 rows, 23 places).

---

## Key Finding: V3 Chip-Level Split Reduces Buffer from 63 % to 7 %

Moving the spatial split from the 1 km mapsheet level (V1/V2) to the 128 px chip level
within each raster (V3) collapses the buffer fraction from ~63 % to ~7–8 %.

The V1/V2 buffer waste was structural: Cluster B (436–437 / 646–648) holds 75.8 % of all
rows in 6 adjacent tiles. Any mapsheet-level test selection from that cluster forces the
5 remaining adjacent tiles (~55 % of all rows) into buffer.  Moving the split boundary
inside each raster dissolves this constraint — every raster contributes training chips.

## Recommendation

**Use E07 (V3, 3×3 blocks, buffer=2 chips)** for all new training runs.

Rationale:
- Buffer=2 chips = 51.2 m > 50 m CWD autocorrelation range (Gu et al. 2024) ✓
- 3×3 blocks gives enough spatial independence per raster ✓
- train% = 69 % vs 15 % for V1 — **4.6× more training data** ✓
- Buffer% = 7.4 % — negligible waste ✓
- Highly stable across seeds (spread < 0.2 pp) ✓

E09 (buffer=1 chip / 25.6 m) gives slightly more training data but is below the empirical
autocorrelation threshold; use only if model performance is confirmed insensitive to leakage.

### Usage

```python
from scripts.spatial_split_experiments._split_v3_chips import write_intra_raster_chip_split
from pathlib import Path

meta = write_intra_raster_chip_split(
    all_candidates=candidates,          # same all_candidates list as model_search_v4

    output_geojson=Path("output/.../split.geojson"),  # optional block map for QGIS
    seed=2026,
    test_fraction=0.20,
    n_blocks=3,         # 3×3 spatial blocks per raster (~333 m × 333 m each)

    balance_cdw_tol=0.20,
)
```

## V1/V2 Mapsheet-Level Split — Structural Buffer Constraint

The 63–65 % mapsheet-level buffer is **irreducible** regardless of algorithm.

| Cluster | Places | Rows | % of total | Intra-cluster adjacency |
|---|---|---|---|---|
| B: 436-437 / 646-648 | 6 | 13 200 | **75.8 %** | Fully connected 2×3 km grid |
| E: 580-582 / 535-543 | 11 | 3 214 | 18.5 % | Two parallel chains |
| Other (A, C, D, F) | 6 | 989 | 5.7 % | Isolated / far apart |

When any place from Cluster B is selected as test (~20 % target), the remaining 5 places
in that cluster (~55 % of all rows) are within 1 tile of at least one test tile and must
be buffered. V3 dissolves this entirely.

## Academic Justification

| Recommendation | Supported by |
|---|---|
| Spatial blocking, not random split | Roberts et al. 2017 (Ecography) |
| Buffer ≥ autocorrelation range (~50 m) | Gu et al. 2024 (Forests); Le Rest et al. 2014 (GEB) |
| Block size is dominant CV parameter | Valavi et al. 2025 (Front. Remote Sens.) |
| High buffer% reflects real autocorrelation cost | Ploton et al. 2020 (Nat. Commun.) |
| Data imbalance (one cluster = 75 % of rows) is a real risk | Pohjankukka et al. 2017 (IJGIS) |
